windows10+github+vs2017配置

Posted on 2018-04-08 | In windows
Words count in article: 490 | Reading time ≈ 2

　　之前使用时自己写得步骤：Windows10+VS2017+Github使用。

  1. 新建项目，勾选增加git版本控制或者项目建好后解决方案右键 add –>the source control；
  2. 左边的窗口中，登陆GitHub账号；
  3. publish界面有两个选项：选择发步到GitHub仓库：输入发布的仓库名称，描述，发布的位置，点击Publish完成；选择发布到远程仓库：登陆GitHub网页，新建仓库，完成后复制https，在VS中输入URL地址，选择发布完成；
  4. 操作完成；
  5. 更新代码后，changes下填写commit注释，commit all 修改本地仓库，commit all and push 修改本地仓库并上传至远程仓库，commit all and sync 修改本地仓库，pull拉取远程仓库到本地仓库更新然后push到远程仓库；
  6. fetch和pull区别：前者只更新了本地commit 的id，后者更新了本地库。

注：全局的配置在C：//users/YOURNAME/中，默认的ssh密钥也在这个目录下

  • GitCmd，一个Terminal 基于Windows的，命令都是win下的shell才能执行。。。（应该和Bash一样的功能)
  • GitGui，Git的一个界面控制，好复杂，不需要使用
  • GitBash，基于Unix的轻量级shell终端，这个里边可以使用vi，挺好用的！！！

ssh–T git@github.com //检查连接正常与否

git branch –a //列出remote远程端和本地仓库的分支

git branch –r //列出remote远程断分支

git config --global user.name “your_username” //设置用户名

git config --global user.email “your_registered_github_Email” //设置邮箱地址

cd ~/.ssh 或cd .ssh //检查本机是否存在ssh key

ssh-keygen -t rsa -C "xxxxxx@yy.com" //密钥制作

git init //在当前目录下初始化git目录

git remote add origin ‘粘贴复制仓库的ssh路径’ //设置远程端的ssh路径

git add . //提交当前目录下所有文件

git commit -m " 加入了***文件" //提交记录说明

git push -u origin master //提交到远程端
